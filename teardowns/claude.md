# Claude（Anthropic）产品深度拆解

> 拆解时间：2025年5月  
> 测试端：Claude.ai Web + iOS App（Pro 账户）  
> 版本：Claude 3.7 Sonnet / Opus  
> 关注点：产品设计哲学 vs OpenAI 的差异、Extended Thinking、Projects 功能

---

## 一、Anthropic 的产品哲学与 OpenAI 的根本差异

理解 Claude 产品之前，必须先理解 Anthropic 和 OpenAI 在使命陈述上的差异：

- **OpenAI**：Build AGI that benefits all of humanity（尽快到达 AGI）
- **Anthropic**：The responsible development and maintenance of advanced AI for the long-term benefit of humanity（以安全为前提的 AI 研发）

这个使命差异**直接体现在产品决策上**：

| 维度 | Claude | ChatGPT |
|------|--------|---------|
| 拒绝策略 | 更详细解释为什么拒绝，提供替代方案 | 更短的拒绝，更少解释 |
| 输出风格 | 更倾向于呈现"我的观点+不确定性" | 更倾向于给出确定性答案 |
| 角色扮演边界 | 严格但有解释 | 更灵活但有时过于宽松 |
| 能力展示 vs 安全 | 宁可看起来"能力弱"也不越边界 | 倾向于展示能力 |

**产品结论**：Claude 的目标用户不是"要最强 AI"的人，而是"要可信赖、有一致性 AI"的人——律师、研究人员、高要求的知识工作者。

## 二、核心功能架构

```
Claude.ai
├── 对话（核心）
│   ├── Claude 3.7 Sonnet（默认，平衡速度/能力）
│   ├── Claude 3 Opus（最强，Pro 专属）
│   └── Extended Thinking（深度推理模式）
├── Projects（知识库 + 持久记忆）
│   ├── 私有文档上传（PDF/文本/代码）
│   ├── Project Instructions（自定义 AI 行为）
│   └── 会话历史在 Project 内共享
├── Artifacts（内容输出容器）
│   ├── 代码（可执行预览）
│   ├── Markdown 文档
│   ├── SVG / HTML（可渲染）
│   └── React 组件（实时预览）
├── 多模态输入
│   ├── 图片理解
│   ├── PDF 阅读（长文档）
│   └── 文件上传
└── API（面向开发者）
    └── Claude API + Prompt Caching + Computer Use
```

## 三、核心功能深度分析

### 3.1 Extended Thinking——产品层面最有意思的决策

Extended Thinking（深度思考模式）是 Claude 3.7 的标志性功能：开启后，模型会先输出一个"思考过程"（用 `<parameter name="content">` 标签包裹，可折叠），再给出最终答案。

**与 OpenAI o1 的设计对比**：
- o1：思维链完全不可见，你只看到最终答案（黑盒）
- Claude 3.7 Thinking：思维链**可选择展示**，用户可以折叠或展开

**为什么这个设计选择值得注意**：
- 可见的思维链 = 更高的透明度 = Anthropic 品牌主张的一致性
- 用户可以判断"AI 是不是想清楚了"，而不只是信任一个答案
- 代价：有些用户看到 AI "想了很多弯路" 反而会降低信任感

**我的判断**：Anthropic 在这里把"透明度"排在"流畅感知"前面，这是一个价值观驱动的产品决策。对于研究/专业用途的用户，透明思维链比光滑的黑盒答案更有价值。

### 3.2 Projects——Claude 对 "Memory" 问题的解答

AI 助手的记忆问题是整个行业都在解决的：ChatGPT 用"记忆"功能（隐式），Claude 用 Projects（显式）。

**Projects 的设计哲学**：
- 用户**主动组织**知识（上传文档、写 Instructions），而不是 AI 自动记忆一切
- 不同 Project 之间完全隔离——"工作 AI"和"个人 AI"不会互相影响
- 会话历史属于 Project，而不是个人账号——清晰的数据归属

**对比 ChatGPT 的隐式记忆**：
- ChatGPT 会自动从对话中提取记忆（"用户喜欢简洁回答"），但你不知道它记了什么、是否记错
- Claude Projects 让用户完全知道 AI "了解"什么——透明原则的又一体现

**实际使用体验**：我把论文阅读放在一个 Project，AI 写作助手放在另一个，两个场景的 AI 行为完全不同（不同的 Instructions），这种隔离感在实际工作中很有价值。

### 3.3 Artifacts——内容输出的容器化

Artifacts 是 Claude 独创的 UX 设计，把生成的内容（代码/文档/HTML）放在一个独立的右侧面板里，而不是混在对话流里。

**为什么这个设计好**：
- 代码/文档和对话分离，不会互相"污染"视觉层次
- Artifacts 可以独立操作（全屏预览、导出、迭代修改）
- 对话继续时，Artifact 自动更新，而不是在聊天里重复粘贴代码

这个设计已经影响了其他产品（Cursor 的 Inline diff、ChatGPT Canvas 都有类似思路），Claude 是最早清晰定义这个 UX 范式的。

### 3.4 Computer Use——目前最超前的 Agent 能力

Claude 3.5 Sonnet 支持 Computer Use API：给 Claude 截图，它能控制鼠标和键盘操作计算机。

**产品层面的判断**：目前还是 Research Preview，并非面向普通用户的产品功能。但它代表了 Anthropic 在"AI 操作计算机"这个范式上的押注——这是从"AI 助手"到"AI Operator"的关键一步。

## 四、对话质量的差异化特征

通过大量测试，总结 Claude 区别于其他模型的输出特征：

### 4.1 不确定性标注

Claude 会主动说"我不确定这个信息是否准确"或"根据我的训练数据截止时间，情况可能已变化"。

这在其他模型里很少见——它们倾向于给出自信的答案（不管是否应该自信）。

**产品影响**：对需要准确性的专业用户（律师/医生/研究人员），这种不确定性标注反而建立了信任。对需要"果断答案"的普通用户，可能会觉得 Claude "不够自信"。

### 4.2 细腻的拒绝方式

Claude 拒绝请求时通常会：
1. 解释拒绝原因
2. 提供可接受的替代方案
3. 询问用户真实意图（有时会发现用户不需要"违规"内容也能解决问题）

相比其他模型的硬拒绝，这种方式降低了误伤率，也体现了"帮助用户"而非"保护自己"的价值取向。

### 4.3 长文档处理能力

200K context window 在实际使用中表现稳定（前提是文档结构清晰）。

我测试过：把 150 页 PDF 的技术规范丢进去，问"第 78 页的表 3 里，A 类设备的最大电流限制是多少"，能准确定位。对比 GPT-4 的"Long Context"，Claude 在长文档中部的信息检索上明显更稳。

## 五、商业模式

| 层级 | 产品 | 价格 | 核心价值 |
|------|------|------|----------|
| 免费 | Claude（有限次） | $0 | 获客 / 产品体验 |
| Pro | Opus + 5x 用量 + Projects | $20/月 | 专业用户 |
| Team | 团队协作 + Projects 共享 | $25/人/月 | 企业小团队 |
| Enterprise | 自定义部署 + SSO + 审计日志 | 定制 | 大企业 |
| API | 按 token 计费 + Prompt Caching | 按量 | 开发者/企业集成 |

**Prompt Caching 的商业逻辑**：对于 System Prompt 很长的 API 用户（比如企业部署场景），缓存 Prompt 可以降低 90% 的成本。这是 Anthropic 对企业用户的重要绑定策略——接入成本越低，迁移意愿越低。

## 七、产品感悟

**最核心的产品观察**：Anthropic 在做一件极其困难的事——用安全 AI 的价值观做商业产品。这导致 Claude 在某些能力测试上看起来"守规矩过头"，但也让它成为企业场景里最受信任的 AI 之一。

**对产品设计的启发**：
1. **品牌一致性的力量**：透明度、不确定性标注、详细的拒绝解释——这些单独看都是小功能，但组合起来构成了极强的品牌认知：Claude 是"诚实的 AI"。
2. **显式 vs 隐式的设计哲学**：Projects（显式记忆）vs ChatGPT 自动记忆（隐式）——两种方式都有市场，但面向不同的用户心智模型。对于想"掌控 AI 了解什么"的专业用户，显式更好。
3. **Extended Thinking 的启示**：把中间过程可视化是建立用户信任的有效手段。我在 Agent 产品设计里也在用这个原则——让用户能看到 Agent 的工作过程，而不只是看结果。

**一句话判断**：Claude 不是在争"最强 AI"，而是在争"最可信 AI"——这个定位在 AI 产品同质化竞争中非常清醒，尤其在企业场景和专业用户群里有显著优势。

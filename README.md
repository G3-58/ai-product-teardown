## 拆解框架

每篇拆解包含以下维度：

```
1. 产品定位与目标用户
2. 核心功能地图
3. 交互设计分析（首屏 / 对话 / 引导）
4. Prompt 工程观察（系统角色、格式控制、兜底策略）
5. 能力边界测试
6. 商业模式与变现路径
7. 与竞品的核心差异
8. 个人产品感悟
```
